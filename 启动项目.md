# 航小票项目启动指南

## 项目概览
当前仓库是整合后的测试版本：
- 后端：`NestJS + TypeORM + SQLite`
- 前端：微信小程序，已整合用户端和发布端页面
- 默认后端地址：`http://localhost:3000`
- 默认管理员账号：`ADMIN2024`
- 默认管理员密码：`HXP-ADMIN`

## 后端启动
进入 [backend](/E:/Study/codes/hangxiaopiao_Test/backend) 后执行：

```bash
npm install
npm run build
npm run start:dev
```

如果需要启动编译产物，使用：

```bash
npm run start:prod
```

服务启动成功后，应能看到 `Nest application successfully started`。

## 后端验证
建议优先验证以下接口：

```bash
curl http://localhost:3000/activities
curl -X POST http://localhost:3000/admin/login -H "Content-Type: application/json" -d "{\"account\":\"ADMIN2024\",\"password\":\"HXP-ADMIN\"}"
curl -X POST http://localhost:3000/users/login -H "Content-Type: application/json" -d "{\"phone\":\"13800138000\",\"password\":\"123456\"}"
```

说明：
- 首次启动会自动创建 `backend/database.db`
- 默认会自动写入管理员账号
- 即使本机没有 Redis 或 RabbitMQ，后端也会降级到同步逻辑，不影响基础联调

## 小程序启动
在微信开发者工具中导入 [frontend](/E:/Study/codes/hangxiaopiao_Test/frontend)。

建议配置：
- 使用测试号或你自己的 AppID
- 在开发环境中勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”
- 确认小程序请求地址仍为 `http://localhost:3000`

小程序主要入口：
- 用户端登录：`pages/login/login`
- 管理员登录：`pages/admin/login/login`
- 角色选择页：`pages/launch/launch`

## 联调顺序
推荐按下面顺序做最短闭环验证：

1. 启动后端并确认 `/activities` 可访问
2. 使用管理员账号登录发布端
3. 创建活动并完成预览发布
4. 使用用户手机号登录
5. 补全资料并查看活动详情
6. 条件允许时继续验证抢票和票根查询

## 常见问题
### 1. 3000 端口被占用
报错：`EADDRINUSE`

处理方式：
- 结束占用 3000 的进程
- 或修改 [backend/.env](/E:/Study/codes/hangxiaopiao_Test/backend/.env) 中的 `PORT`

### 2. 后端能编译但无法启动
优先确认是否使用了错误的编译入口。当前编译产物入口是：

```text
dist/src/main.js
```

### 3. 小程序提示网络错误
优先检查：
- 后端是否真的在运行
- 微信开发者工具是否关闭域名校验
- 小程序 API 地址是否仍指向 `http://localhost:3000`

### 4. 页面出现乱码
当前仓库历史上存在编码损坏。若发现新页面乱码，优先检查：
- 文件是否为 UTF-8 编码
- 是否有历史错误转码内容被继续复制
- 终端输出是否为 UTF-8
